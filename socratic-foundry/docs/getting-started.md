# Socratic Foundry 왕초보 가이드 — 설치부터 PRD까지

> 이 문서는 Claude Code 를 처음 쓰는 분도 따라올 수 있게 썼습니다.
> 순서대로 읽으면서 그대로 하시면 됩니다. 예상 소요: 설치 5분 + 인터뷰 30~45분.

---

## 1. 이 스킬은 무엇인가요?

**한 줄로**: 아이디어를 들고 오면, AI 가 소크라테스처럼 캐물어서 "정말 만들 가치가 있는지"를 검증하고, 마지막에 **기획 보고서와 PRD(개발 명세서)**까지 만들어 주는 도구입니다.

보통의 AI 는 "다이어트 앱 만들어줘"라고 하면 바로 기능 목록을 뽑아줍니다.
이 스킬은 반대로 갑니다 — **먼저 의심합니다.**

- "그 아이디어, 어떤 실제 경험에서 나왔나요?"
- "그 문제로 실제로 돈이나 시간을 잃은 적이 있나요?"
- "당신조차 돈을 안 냈던 문제에, 남들은 돈을 낼까요?"

이 질문들을 브라우저 화면에서 하나씩 답하다 보면, 막연했던 아이디어가
**증거가 붙은 문제 → 검증 실험 → 개발 명세서(PRD)**로 바뀝니다.

### 끝나면 받는 것

1. **기획 보고서** — 인터뷰 전체를 정리한 HTML 문서 (PDF 저장 가능)
2. **One-page 요약** — 핵심만 담은 한 장짜리 문서
3. **검증 실험 계획** — 개발 전에 7~14일 안에 해볼 수 있는 실험
4. **PRD (제품 요구사항 문서)** — 기능·화면·데이터·개발 작업 목록까지 담긴 **개발 착수용 명세서**. 이 파일 하나를 개발자나 AI 코딩 도구에 주면 개발이 시작됩니다.

## 2. 준비물

- **Claude Code** — Anthropic 의 AI 코딩 도구입니다. 터미널(명령어 창)에서 실행됩니다.
  - 설치가 안 되어 있다면: https://claude.com/claude-code 안내를 따라 설치하세요.
- **Python 3** — 인터뷰 화면을 띄우는 데 씁니다.
  - Mac 은 기본으로 들어 있습니다. 확인하려면 터미널에 `python3 --version` 을 입력하세요. 버전 번호가 나오면 준비 끝.
- **웹 브라우저** — Chrome, Safari 등 아무거나.

그 외 설치할 것은 **없습니다.** 이 스킬은 외부 프로그램이나 유료 API 를 쓰지 않습니다.

## 3. 설치하기 (5분)

받으신 ZIP 파일(`socratic-foundry-*.zip`) 기준으로 설명합니다.

### 방법 A — 폴더째 복사 (가장 쉬움, 추천)

터미널을 열고 아래를 한 줄씩 입력하세요. (ZIP 이 다운로드 폴더에 있다고 가정)

```bash
cd ~/Downloads
unzip socratic-foundry-*.zip
mkdir -p ~/.claude/skills
cp -R socratic-foundry ~/.claude/skills/
```

### 방법 B — 심링크 (원본 폴더를 다른 곳에 두고 싶을 때)

```bash
cd ~/Downloads
unzip socratic-foundry-*.zip
mv socratic-foundry ~/원하는/보관/위치/
cd ~/원하는/보관/위치/socratic-foundry
bash scripts/install-symlink.sh
```

### 설치 확인

1. Claude Code 를 **재시작**합니다 (켜져 있었다면 껐다 켜기).
2. 아무 폴더에서 Claude Code 를 열고 `/socratic-foundry` 라고 입력해 보세요.
3. 스킬이 반응하면 설치 성공입니다.

> ⚠️ 안 되면: `ls ~/.claude/skills/socratic-foundry/SKILL.md` 를 입력해 보세요.
> "No such file" 이 나오면 폴더가 제 위치에 없는 것입니다. 3번 과정을 다시 확인하세요.

## 4. 시작하기

작업할 폴더(프로젝트 폴더)에서 Claude Code 를 열고, 아이디어와 함께 스킬을 부릅니다:

```
/socratic-foundry 다이어트 다이어리: 사진으로 칼로리를 자동 계산하는 웹앱을 만들고 싶어
```

아이디어가 아직 없어도 됩니다:

```
/socratic-foundry 내 경험으로 뭘 만들지 모르겠어
```

몇 초 뒤 **브라우저 창이 자동으로 열립니다.** 여기서부터 모든 진행은 브라우저에서 합니다.
터미널은 그냥 켜 두세요 — Claude 가 뒤에서 당신의 답을 받아 다음 화면을 만듭니다.

## 5. 첫 화면 — 인터뷰 계약

브라우저 첫 화면에서 정하는 것들입니다.

### 모드 고르기 (5개 중 1개)

- **Quick Scan** (10~15분) — 시간이 없을 때. 핵심 문제·자산 후보만 빠르게 훑습니다.
- **Standard Discovery** (30~45분) — **처음이라면 이걸 고르세요.** 문제 발굴부터 기회 포트폴리오, 검증 실험까지 전부 돕니다.
- **Deep Mining** (60~90분+) — 시간을 들여 깊게. 논박·취약점 감사까지 포함됩니다.
- **Idea Challenge** — 이미 아이디어가 확고할 때. "이 아이디어를 반박해 봐" 모드입니다.
- **Domain Productization** — 특정 분야 경력자가 "내 지식을 상품으로" 만들 때.

### 나머지 입력칸

- **최초 진술** — 만들고 싶은 것을 자유롭게. 스킬 호출 때 적었다면 미리 채워져 있습니다.
- **오늘 가장 알고 싶은 것** — 예: "이 앱, 만들 가치가 있는지"
- **질문 깊이 / 금지 영역** — 답하기 불편한 주제가 있으면 적으세요. 그 영역은 묻지 않습니다.
- **인터뷰 후 내릴 결정** — 예: "개발할지 말지 결정한다"

다 적고 시작 버튼을 누르면 인터뷰가 시작됩니다.

## 6. 인터뷰 진행 요령

### 기본 리듬

화면에 **질문이 하나씩** 나옵니다. 질문마다 "왜 이걸 묻는지" 이유가 함께 표시됩니다.
예상 답변 버튼 중에 고르거나, 직접 입력하세요. 답하면 Claude 가 잠시 생각한 뒤(화면 상단에 "생각 중" 배너) 다음 질문을 만듭니다.

### 화면의 보조 버튼들

- **더 깊게** — 방금 주제를 한 겹 더 파고들고 싶을 때
- **건너뛰기** — 이 질문은 넘어가기 (공백으로 기록됩니다)
- **예시 보기** — 어떻게 답해야 할지 감이 안 올 때 답의 형식 예시를 보여줍니다
- **잘 모르겠습니다** — 솔직한 "모름"도 중요한 데이터입니다. 억지로 답하지 마세요
- **이전 답변 수정** — 방금 답을 고치고 싶을 때
- **이 단계 정리하기 →** — 단계가 충분해지면 이 버튼으로 다음 단계로 넘어갑니다

### 잘 답하는 팁 3가지

1. **솔직하게.** "돈 쓴 적 없어요", "그런 경험 없어요"가 부끄러운 답이 아닙니다 — 오히려 가장 값진 데이터입니다. 이 스킬은 듣기 좋은 말을 하지 않는 대신, 거짓 확신도 주지 않습니다.
2. **구체적인 장면으로.** "힘들어요"보다 "지난주 화요일 회식 때 삼겹살 칼로리를 못 찾아서 기록을 포기했어요"가 백배 유용합니다.
3. **모순 지적을 환영하세요.** 앞뒤 안 맞는 답을 하면 스킬이 짚어냅니다. 화내는 게 아니라, 진짜 생각을 찾아주는 과정입니다.

### 중간에 나오는 특별한 화면들

- **지식 그래프** — 지금까지 발견한 것을 트리 그림으로 보여줍니다. 노드마다 "맞다/부분적으로 맞다/근거 필요/예외 있음/더 파보자" 피드백을 줄 수 있습니다. 다 확인했으면 승인 버튼.
- **문제 재정의** — "처음 하신 말 → 관찰된 문제 → 더 깊은 문제 → 사업 기회" 4단 비교가 나옵니다. **당신이 승인해야만** 다음으로 갑니다.
- **기회 포트폴리오** — 서로 다른 시간축의 기회 5개가 카드로 나옵니다. 각 카드에 증거 등급(A~E)이 붙어 있습니다. D·E 는 "아직 가설"이라는 뜻이니 등급을 꼭 보세요. 마음에 드는 것을 선택하세요.
- **검증 실험** — 개발 없이 7~14일 안에 해볼 수 있는 실험들이 나옵니다. 하나를 고르면 마무리로 갑니다.

## 7. 마지막 화면 — 보고서와 PRD

인터뷰가 끝나면 최종 화면에 산출물 목록이 나옵니다.

- **report.html** — 종합 기획 보고서. 열어서 오른쪽 아래 "인쇄 / PDF 저장" 버튼을 누르면 PDF 가 됩니다. 보고서 안에 **PRD 요약과 "다음 단계" 안내**가 들어 있습니다.
- **prd.md** — ⭐ 가장 중요한 파일. 기능 목록(우선순위·완료 기준 포함), 화면 명세, 데이터 모델, **Phase 별 개발 작업 목록(백로그)**까지 담긴 개발 착수용 명세서입니다.
- one-page-brief.md, validation-plan.md 등 — 요약본과 실험 계획서.

모든 파일은 작업 폴더의 `socratic-foundry-sessions/날짜-이름/outputs/` 안에 저장됩니다.
어디에도 업로드되지 않습니다 — 전부 당신 컴퓨터 안에만 있습니다.

## 8. PRD 를 받았다 — 이제 뭘 하나요?

PRD 맨 앞에는 **"개발 착수 조건"**이 있습니다. 보통 "검증 실험을 먼저 통과하라"는 내용입니다.
이걸 건너뛰고 바로 개발하는 것은 이 스킬이 가장 말리는 일입니다. 실험부터 하세요 — 대부분 개발 없이, 돈 거의 안 들이고 할 수 있게 설계되어 있습니다.

실험을 통과했다면, 두 갈래 중 하나로 갑니다.

### 경로 A — PRD 로 바로 개발 (기본, 별도 도구 불필요)

`prd.md` 파일 하나를 개발자에게 주거나, AI 코딩 도구(Claude Code 등)에 열어 주고 이렇게 말하면 됩니다:

```
이 PRD 의 §9 개발 백로그 Phase 1 을 순서대로 구현해줘.
§5 의 완료 기준과 §8 의 비기능 요구사항을 지켜줘.
```

PRD 는 이 지시 한 줄로 개발이 시작되도록 만들어져 있습니다.

### 경로 B — 스킬팩이 있다면 (선택)

vibelabs 기획 스킬팩(/screen-spec, /tasks-generator, /auto-orchestrate)이 설치된 환경이라면,
PRD 를 그 파이프라인에 넣어 화면 명세 → 태스크 분해 → 자동 빌드로 이어갈 수 있습니다.
없어도 전혀 문제 없습니다 — 경로 A 가 기본입니다.

## 9. 자주 묻는 질문 (FAQ)

**Q. 중간에 브라우저나 터미널을 꺼버렸어요.**
A. 괜찮습니다. 모든 진행 상황이 `socratic-foundry-sessions/` 폴더의 state.json 에 저장됩니다.
같은 폴더에서 `/socratic-foundry` 를 다시 부르면 "이어하기" 를 물어봅니다.

**Q. 브라우저가 자동으로 안 열려요.**
A. 터미널에 표시된 주소(보통 `http://127.0.0.1:8930`)를 브라우저 주소창에 직접 입력하세요.

**Q. 답을 보냈는데 화면이 안 바뀌어요.**
A. Claude 가 생각하는 중입니다(상단 배너 확인). 30초 이상 멈춰 있으면 터미널의 Claude Code 에 "계속 진행해줘"라고 입력해 보세요.

**Q. 질문이 너무 아픈 곳을 찔러요.**
A. 정상입니다 — 그게 이 스킬의 일입니다. 다만 답하기 싫은 영역은 "건너뛰기"를 누르거나, 처음 계약 화면의 "금지 영역"에 적어두면 묻지 않습니다.

**Q. 결과가 "만들지 마세요"에 가깝게 나왔어요.**
A. 그것도 성공입니다. 몇 달의 개발과 비용을 아낀 것입니다. 보고서의 "가장 위험한 가설"과 실험 계획을 보세요 — 검증에서 살아남으면 그때 만들면 됩니다.

**Q. 증거 등급 A~E 가 뭔가요?**
A. 각 기회(사업 아이디어)가 얼마나 검증됐는지입니다.
A = 실제 결제까지 확인됨 / B = 실제 문제·행동 확인 / C = 사례는 있으나 지불 미확인 / D = 추론뿐 / E = 근거 없음.
D·E 가 나쁜 게 아니라, "검증이 먼저"라는 신호입니다.

**Q. [사실] [추론] [가설] 표식은 뭔가요?**
A. 이 스킬의 핵심 규율입니다. 당신이 말한 것(사실), AI 가 연결해 낸 것(추론), 아직 확인 안 된 것(가설)을 절대 섞지 않습니다. 보고서를 읽을 때 이 표식을 보면 어디까지 믿어도 되는지 알 수 있습니다.

**Q. 인터뷰 내용이 외부로 전송되나요?**
A. 인터뷰 화면과 저장 파일은 전부 로컬(당신 컴퓨터)입니다. Claude Code 자체의 AI 통신 외에 이 스킬이 따로 데이터를 보내는 곳은 없습니다.
